# 前端技术与Node.js

HTML = 结构与内容（数据载体）

CSS = 样式与布局（视觉表现）

JavaScript = 逻辑与交互（行为控制）

## 1、HTML

**超文本标记语言（HyperText Markup Language）**

**定位**：**结构化文档描述语言**，用于定义页面的**语义结构与内容载体**。

**本质**：标记型、声明式、无逻辑、无计算能力。

**核心职责**：

1. 定义页面包含哪些**元素节点**（标签）。
2. 为内容赋予**语义**（标题、段落、按钮、输入框、列表、容器等）。
3. 提供元素**属性**（id、class、src、type 等）作为标识与配置。
4. 构建**DOM 树结构**（Document Object Model），为后续渲染与操作提供结构基础。

**语法特征**：

- 由`<标签>`构成，标签可嵌套、可自闭和。

- 不支持变量、条件、循环、函数。

- 仅描述 “**有什么**”，不描述 “**什么样**” 和 “**做什么**”。

## 2、CSS

层叠样式表（Cascading Style Sheets）

  **定位**：**表现层描述语言**，用于定义 HTML 元素的**视觉呈现与布局规则**。

  **本质**：样式型、声明式、无计算逻辑（现代 CSS 具备有限表达式）。

  **核心职责**：

  1. 定义元素**布局模型**（盒模型、Flex、Grid、定位、浮动）。
  2. 定义**视觉样式**（颜色、尺寸、字体、边距、背景、边框、阴影等）。
  3. 定义**状态与响应式**（hover、active、媒体查询、动画过渡）。
  4. 通过**选择器**匹配元素，遵循**层叠、优先级、继承**规则应用样式。

  **语法特征**：

  - `选择器 { 属性: 值; }`
  - 不直接操作数据，不执行业务逻辑。
  - 仅描述 “**长什么样**”，不描述 “**做什么**”。

## 3、JavaScript（JS）

动态脚本语言

**定位**：**行为层与逻辑层编程语言**，实现交互、数据处理、系统调用。

**本质**：解释型、动态类型、基于原型、支持函数式 / 面向对象。

**核心职责**：

1. 操作 DOM：增删改查元素、修改内容、绑定事件。
2. 实现**交互逻辑**：点击、输入、表单验证、状态管理。
3. 处理**数据与计算**：变量、条件、循环、函数、异步（Promise/async）。
4. 调用**系统 / 平台能力**：网络请求、文件读写、USB / 串口 / 蓝牙（Node.js/Electron 环境）。
5. 驱动应用生命周期与状态流转。

**语法特征**：

- 支持完整编程范式：变量、表达式、控制流、函数、对象、模块。
- 可与 HTML/CSS 完全分离，也可内联。
- 负责 “**做什么、怎么交互、如何通讯**”。

**运行环境**：

- 浏览器
- Node.js
- Electron
- 小程序、APP 引擎等

## TypeScript（TS）

**定义**：JavaScript 的**超集语言**，基于 ECMAScript 扩展，增加**静态类型系统**。

**定位**：**带类型校验的 JS 增强版**。

**本质**：

- 不是新语言

- 不能直接运行

- 必须编译成 JS 才能执行

  

  核心扩展：

- 静态类型

- 接口 interface

- 类型注解

- 泛型

- 枚举

  最终产物

  ：编译后 = 普通 JavaScript。

## Node.js

**定义**：**基于 V8 引擎的 JavaScript 运行时环境（Runtime）**。

**定位**：**让 JS 可以脱离浏览器运行在操作系统上**。

**核心提供**：

- 文件系统 fs

- 网络 net /http

- 进程 process

- 子进程 child_process

- 串口、USB、蓝牙调用能力

- 模块系统

  

  本质：

  运行环境 + 系统 API 集合。

# Electron 

## 1、Electron 官方定义

**Electron 是一个由 GitHub 开发的开源框架。**

它允许开发者使用 **Web 技术栈（HTML + CSS + JavaScript）** 构建 **跨平台桌面应用程序**。

Electron 合并了两大核心组件：

1. **Chromium**：负责渲染界面（浏览器内核）
2. **Node.js**：负责操作系统级 API 调用

---

## 2、Electron 核心架构

Electron 应用由 **2 类进程** 构成：

### 1. 主进程 Main Process

- 唯一
- 基于 Node.js
- 负责：创建窗口、控制菜单、系统托盘、硬件通讯、自动更新、后台任务
- **可以调用所有 Node.js API（文件、USB、串口、蓝牙、网络）**

### 2. 渲染进程 Renderer Process

- 每个窗口一个
- 基于 Chromium
- 负责：显示界面（HTML/CSS/JS）
- 默认不能调用 Node.js（可开启支持）

### 3. 通信机制 IPC

主进程 ↔ 渲染进程

通过 **IPC（进程间通信）** 传递数据。

---

## 3、Electron 的能力

### ✅ 桌面窗口管理

- 创建窗口、全屏、置顶、最小化、无边框窗口

### ✅ 系统原生功能

- 托盘图标、消息通知、快捷键、开机自启

### ✅ 操作系统底层能力（Node.js）

- 文件读写
- 进程管理
- 网络请求
- **串口、USB、蓝牙、HID 硬件通讯**
- 打印机、扫描设备

### ✅ 强大的 Web 界面

- 任何前端框架（Vue / React / Angular）
- 本地 HTML 文件
- 在线网页

### ✅ 一次开发，三平台发布

- Windows → exe / msi
- macOS → dmg
- Linux → deb / rpm

---

## 4、Electron 运行流程

1. 启动 `main.js`（主进程入口）
2. 主进程创建 `BrowserWindow` 窗口
3. 窗口加载 `index.html`
4. 渲染进程渲染界面
5. 主进程与渲染进程通过 IPC 通信
6. 可调用 Node.js 操作硬件 / 系统

---

## 5、Electron 开发技术栈

1. **HTML**：结构
2. **CSS**：样式
3. **JavaScript**：逻辑
4. **Node.js**：系统 / 硬件 API
5. **Electron API**：窗口、菜单、托盘等

**不需要 C++ / C# / Java**

## 6、最简 Electron 项目结构

```
your-app/
├── package.json  项目配置
├── main.js       主进程
└── index.html    界面
```

**package.json**

```json
{
  "name": "app",
  "main": "main.js"
}
```

**main.js**

```javascript
const { app, BrowserWindow } = require('electron')

app.whenReady().then(() => {
  const win = new BrowserWindow({
    width: 800,
    height: 600
  })
  win.loadFile('index.html')
})
```

**index.html**

```html
<h1>我的 Electron 桌面软件</h1>
```

## 7、如何运行 & 打包

### 运行

```bash
npm install electron
npx electron .
```

### 打包成 exe

```bash
npm install @electron/packager -D
npx electron-packager . myapp --platform=win32
```

自动输出 `.exe` 可分发文件。

## 8、Electron 优点/缺点

### 优点

- 开发效率极高
- 跨平台
- 界面美观现代化
- 可直接调用硬件（USB / 蓝牙 / 串口）
- 生态极其强大
- 适合快速开发商业工具

### 缺点

- 应用体积较大（至少 50MB+）
- 内存占用高于原生软件
- 极高性能场景不如 C++
